What ArbiterX Does

① Codebase Map

Parses your code with tree-sitter, ranks symbols with PageRank, and sends only what the AI needs. 200,000 tokens → 5,000 tokens.

② Smart Router

Routes trivial tasks to cheap models (Haiku) and complex tasks to powerful ones (Opus). Same quality, fraction of the cost.

③ Quality Gate

Scores every piece of AI-generated code 0–100. Checks security, robustness, efficiency, types, style, and completeness. Rejects below 70.

Quick Start

# Install
pip install arbiterx

# Initialize (once per project)
arbiterx init
arbiterx map              # Index your codebase (~30 seconds)

# Use
arbiterx route "task"     # See model routing decision
arbiterx query Symbol     # Find any function instantly
arbiterx gate --file f.py # Score any file 0-100

The 10 Engineering Rules

These run on every piece of generated code. No exceptions.

 1. YAGNI           — Don't build features nobody asked for
 2. Error Handling   — No bare except, no swallowed errors
 3. Type Safety      — Type all function signatures
 4. Resource Cleanup — Use context managers, never bare open()
 5. No Magic Numbers — Name your constants
 6. No Dead Code     — No commented code, no unused imports
 7. Single Purpose   — One function = one job (max 30 lines)
 8. Fail Fast        — Validate inputs at function entry
 9. Idempotency      — Operations safe to retry
10. Performance      — No nested loops when avoidable

Frequently Asked Questions

What is ArbiterX?

ArbiterX is intelligent middleware for AI coding assistants. It sits between you and your AI tool (Claude Code, Codex, Cursor, Copilot) and does three things: compresses context by 97% so the AI sees only what matters, routes tasks to the right model based on complexity, and scores all generated code with a quality gate (0-100).

How does ArbiterX reduce tokens by 97%?

It parses your codebase with tree-sitter, extracts function and class definitions, ranks them by importance using PageRank, and stores them in SQLite. When the AI needs context, it gets relevant signatures (5,000 tokens) instead of full files (200,000 tokens). Measured on real queries against its own 46-file codebase.

Does it slow down my workflow?

No. The codebase map builds in under 1 second. Symbol queries take 0.5ms. The quality gate uses regex-based static analysis — no LLM calls needed for scoring.

Does ArbiterX work offline?

Yes. The map, router, and quality gate all run locally. Only the LLM adapters need internet (use Ollama for fully offline AI generation).

Is ArbiterX free?

Yes. Apache-2.0 license. Zero telemetry. Your code never leaves your machine.

How is this different from a linter?

Linters check syntax. ArbiterX enforces engineering decisions — YAGNI, proper abstractions, security patterns, and architectural discipline. It also handles context compression and model routing, which linters don't touch.

What programming languages does ArbiterX support?

22 languages: Python, TypeScript, JavaScript, Go, Rust, Java, C, C++, C#, Ruby, PHP, Swift, Kotlin, Scala, Elixir, Haskell, OCaml, Lua, Zig, Bash, JSX, and TSX.

What does the quality gate check?

Security (hardcoded secrets, SQL injection, eval/exec), Robustness (error handling, resource cleanup), Efficiency (O(n²) patterns, string concat in loops), Type Safety (typed signatures), Style (clean naming), and Completeness (no TODOs). Code scoring below 70/100 is rejected.

How does smart routing save money?

Tasks are classified by complexity. A variable rename goes to Haiku ($0.25/M tokens), a bug fix to Sonnet ($3/M), and architecture redesign to Opus ($15/M). You get the same quality at a fraction of the cost because most tasks are simple.

What LLM providers does ArbiterX support?

Anthropic (Claude Opus, Sonnet, Haiku), OpenAI (GPT-4o, o1, o3), Google (Gemini 2.0 Pro/Flash), Ollama (any local model), and OpenRouter (all models via unified API).
